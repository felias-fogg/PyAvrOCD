How to Generate a New Version

  1. Inside pyavrocd:
  2. Bump version number in pyproject.toml
  3. poetry install
  4. git commit -a -m ... (this calls pre-commit!)
  5. git push
  6. poetry build (for PyPi)
  7. poetry publish (also for PyPi)
  8. For each macOS, Linux-Arm, Windows, macOS-Parallels, Linux-on-PC,
  9. run local: newversion.sh
  10. This will populate the directories in felias-fogg.github.io:
    • felias-fogg.github.io/binaries/
  11. Create packages and assets:
  12. Change into felias-fogg.github.io/avrocd-tools
  13. Run ./packer.sh, which will populate the folders:
    • The avrocd-tools folder itself with tools packages
    • The assets folder with binary tars of dw-gdbserver
  14. git add \<new tool packages> in avrocd-tools
  15. git commit ...
  16. git push
  17. Create new dw-gdbserver release:
  18. Goto GitHub repo pyavrocd
  19. Create new release
  20. Include as assets the new binaries from felias-fogg.github.io/assets
  21. Change into xCore
  22. Change version number for AVROCDTOOLS_VERSION in Add_avrocd-tools-release.sh
  23. Change AVROCDTOOLSVERSION number in Boads_manager_release.sh
  24. create PR
